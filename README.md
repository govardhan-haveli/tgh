# Goverdhan Haveli - Janmashtami Mahotsav Web App

A divine, responsive, feature-packed web application for **Goverdhan Haveli** to celebrate the annual **Shree Krishna Janmashtami Mahotsav**.

## 🌟 Key Features

1. **Janmashtami Countdown Timer**: Live countdown timer calculating remaining Days, Hours, Minutes, and Seconds until Janmashtami.
2. **Matching T-Shirt Registration**: Group members form to reserve uniform T-shirts (Name, Mobile Number, T-Shirt Size dropdown).
3. **Supabase Database Integration**: Stores T-shirt registrations directly in Supabase DB with an automatic LocalStorage fallback mode for instant offline demo.
4. **Password-Protected `/admin` Route**:
   - Access restricted by passwords defined in `data.js`: `Govardhan-Haveli-2026` & `TGH@2026`.
   - Admin view to Accept, Reject, Mark Delivered, or Delete registrations.
   - Search & Filter by Name, Mobile Number, Size, and Status.
   - Size breakdown analytics for placing bulk wholesale T-shirt printing orders.
   - One-click **Export to CSV** button.
5. **Media Galleries**: Celebration photo gallery with category filter tabs and lightbox modal, plus embedded celebration video showcase player.
6. **Cloudinary Setup**: Prepared for future direct image upload integration.
7. **GitHub Pages Ready**: Configured with `gh-pages` and `HashRouter` for zero-404 deployment on `github.io`.

## ⚡ Supabase Setup Instructions

1. Log into your [Supabase Dashboard](https://supabase.com/).
2. Create a new project or select your existing project.
3. Go to **SQL Editor** -> **New Query** and run the following script:

```sql
CREATE TABLE tshirt_registrations (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  name TEXT NOT NULL,
  mobile TEXT NOT NULL,
  size TEXT NOT NULL,
  status TEXT DEFAULT 'Pending' NOT NULL
);

-- Enable Row Level Security (RLS)
ALTER TABLE tshirt_registrations ENABLE ROW LEVEL SECURITY;

-- Add policies for public anonymous access
CREATE POLICY "Allow anonymous read access" ON tshirt_registrations FOR SELECT USING (true);
CREATE POLICY "Allow anonymous insert access" ON tshirt_registrations FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow anonymous update access" ON tshirt_registrations FOR UPDATE USING (true);
CREATE POLICY "Allow anonymous delete access" ON tshirt_registrations FOR DELETE USING (true);
```

4. Copy your project **API URL** and **anon Key** from **Project Settings** -> **API**.
5. Paste them into `.env`:
   ```env
   VITE_SUPABASE_URL=https://your-project.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-key
   ```

## 🚀 Development & GitHub Pages Deployment

### Run Locally
```bash
npm run dev
```

### Build Production Bundle
```bash
npm run build
```

### Deploy to GitHub Pages (`github.io`)
```bash
npm run deploy
```
*Note: Make sure your `package.json` "homepage" field points to your GitHub Pages repository URL.*

## ☁️ Cloudinary Image Upload Setup

To enable direct Cloudinary image uploading in the future:
1. Create a Cloudinary account.
2. Create an **Unsigned Upload Preset** in Cloudinary Settings -> Upload.
3. Add credentials to `.env`:
   ```env
   VITE_CLOUDINARY_CLOUD_NAME=your-cloud-name
   VITE_CLOUDINARY_UPLOAD_PRESET=your-preset-name
   ```
